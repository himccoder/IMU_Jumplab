IMU Jumplab — Data Collection Guide
Step-by-step: Hardware setup → Flash → Collect → Verify
======================================================================

PART 1 — ONE-TIME SETUP (do this once before any session)
----------------------------------------------------------------------

STEP 1: Install ESP-IDF
  1. Go to: https://docs.espressif.com/projects/esp-idf/en/stable/esp32/get-started/
  2. Download and run the Windows Installer (ESP-IDF Tools Installer).
  3. It installs everything: toolchain, CMake, Python, idf.py.
  4. When done, open "ESP-IDF 5.x CMD" from the Start Menu (use this
     terminal for ALL firmware commands below).

STEP 2: Wire the MPU6050 to the ESP32
  Use this exact wiring:

    MPU6050 Pin   ->   ESP32 Pin
    -----------        ---------
    VCC           ->   3.3V        (NOT 5V — MPU6050 is 3.3V only)
    GND           ->   GND
    SDA           ->   GPIO 1
    SCL           ->   GPIO 0
    AD0           ->   GND         (sets I2C address to 0x68)
    INT           ->   not connected

  !! IMPORTANT — GPIO 0 boot note !!
  GPIO 0 is the ESP32 boot mode pin. If the board fails to boot
  (stuck in download mode), do this:
    a) Unplug the USB cable.
    b) Disconnect the SCL wire from GPIO 0.
    c) Plug USB back in — board boots normally.
    d) Reconnect SCL wire to GPIO 0.
  This is a one-time annoyance. After first successful boot it
  stays stable.

STEP 3: Install Python dependencies (do once)
  Open a normal terminal (PowerShell or CMD) in the project folder:
    pip install -r requirements.txt


PART 2 — FLASH THE FIRMWARE (do once, or after firmware changes)
----------------------------------------------------------------------

STEP 4: Open the ESP-IDF terminal
  - Start Menu -> "ESP-IDF 5.x CMD"
  - Navigate to the firmware folder:
      cd C:\Users\himni\Documents\IMU_Jumplab\firmware

STEP 5: Set the target chip
  If using a standard ESP32:
      idf.py set-target esp32

  If using ESP32-C3 or ESP32-S3:
      idf.py set-target esp32c3
      idf.py set-target esp32s3

STEP 6: Build the firmware
      idf.py build
  Expected output: "Project build complete."
  This takes 1-2 minutes the first time (compiles the full IDF).

STEP 7: Find your COM port
  - Plug the ESP32 into USB.
  - Open Device Manager (Win+X -> Device Manager).
  - Expand "Ports (COM & LPT)".
  - Look for "Silicon Labs CP210x" or "CH340" — note the COM number
    (e.g., COM3 or COM7).
  OR run from a normal PowerShell:
      python main.py collect --list-ports

STEP 8: Flash and verify
  Back in the ESP-IDF CMD terminal:
      idf.py -p COM3 flash monitor
  Replace COM3 with your actual port.

  Expected serial monitor output:
      I (xxx) imu: === IMU Jumplab Firmware  v1.0 ===
      I (xxx) imu: I2C initialised  SDA=GPIO1  SCL=GPIO0
      I (xxx) imu: MPU6050 detected  WHO_AM_I=0x68
      I (xxx) imu: MPU6050 configured  range=+/-8g  scale=4096 LSB/g
      timestamp_ms,ax,ay,az
      312,0.0123,-0.0089,9.7821
      322,0.0145,-0.0091,9.7834
      ...

  If you see "MPU6050 not found!" -> recheck wiring (VCC/GND/SDA/SCL).
  Press Ctrl+] to exit the monitor WITHOUT stopping the firmware.


PART 3 — COLLECTING DATA (repeat for each session)
----------------------------------------------------------------------

STEP 9: Mount the sensor
  - Secure the ESP32+MPU6050 firmly to your lower back / waist
    using Velcro straps or a 3D-printed belt clip.
  - The sensor board should lie FLAT (z-axis pointing up/down).
  - Run the USB cable to your laptop — keep slack so it doesn't
    yank the sensor during jumps.
  - Shake the device — it should NOT wobble independently of your body.

STEP 10: Open a PowerShell terminal in the project folder
      cd C:\Users\himni\Documents\IMU_Jumplab

STEP 11: Record an idle calibration session (30 seconds)
  Stand perfectly still for the full recording.
      python main.py collect --port COM3 --output data/idle_calib.csv --duration 30
  This gives us the gravity baseline for threshold calibration.

STEP 12: Record a walking session (60 seconds)
  Walk normally for the full recording.
      python main.py collect --port COM3 --output data/walk_session.csv --duration 60

STEP 13: Record a jump session (60-90 seconds, 10+ jumps)
  Do the following sequence while recording:
    - 5 seconds: stand still
    - 3 jumps: small hops (~20 cm)
    - 5 seconds: stand still
    - 3 jumps: medium height (~35 cm)
    - 5 seconds: stand still
    - 3 jumps: max effort
    - 5 seconds: stand still
  Command:
      python main.py collect --port COM3 --output data/jump_session_01.csv --duration 90


PART 4 — VERIFY AND INSPECT DATA
----------------------------------------------------------------------

STEP 14: Process and detect jumps
      python main.py process --input data/jump_session_01.csv --plot --save-report

  Check the printed report:
    - Did it detect the correct number of jumps?
    - Do the heights look physically reasonable?
    - Open data/reports/04_jump_events.png to visually confirm.

STEP 15: Calibrate thresholds if needed
  If jumps are missed (freefall threshold too tight):
      python main.py process --input data/jump_session_01.csv --freefall-g 0.40 --plot

  If walking triggers false positives (threshold too loose):
      python main.py process --input data/walk_session.csv --freefall-g 0.30 --plot
      (should report 0 jumps detected)

  Tune --freefall-g between 0.25 and 0.50 until:
    - All real jumps are detected
    - Walking/running produces 0 false positives

STEP 16: Label your data for the ML classifier
  Open the CSV in Excel or a text editor.
  Add a column called "label" with values:
    idle, walk, run, jump, freefall
  Match the label to what was happening at each timestamp.
  Save as:  data/labelled_session_01.csv

STEP 17: Train the classifier on real data
      python main.py classify --input data/labelled_session_01.csv

  Target: F1-macro > 0.90 across all classes.


QUICK REFERENCE — Common Commands
----------------------------------------------------------------------
  List serial ports:
      python main.py collect --list-ports

  Record 30s session:
      python main.py collect --port COM3 --output data/session.csv --duration 30

  Process a recording (no plots):
      python main.py process --input data/session.csv

  Process + show plots + save PNGs:
      python main.py process --input data/session.csv --plot --save-report

  Train ML classifier:
      python main.py classify --input data/labelled_session_01.csv

  Full pipeline test (no hardware needed):
      python main.py simulate
      python main.py process --input data/simulated/jump_session.csv --plot
